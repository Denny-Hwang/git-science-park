# 개발 로드맵 (Roadmap)

> Git Science Park — 과학사 인터랙티브 실험실 (Interactive Science History Lab)
> 과학사의 핵심 실험 **53개**를 "조작 → 관찰 → 해석"으로 직접 다루며 배우는 인터랙티브 학습 플랫폼.

이 문서는 53개 실험을 **10주(6 Phase)**에 걸쳐 구현·배포하는 전체 개발 계획을 정의한다. 모든 실험은 순수 정적 웹(HTML5 + CSS3 + Vanilla JavaScript, Canvas/SVG, **Zero Dependencies**)으로 제작하며, `main` 브랜치 push 시 GitHub Actions(`.github/workflows/deploy.yml`)가 `src/` 폴더를 GitHub Pages로 자동 배포한다.

- 배포 URL: <https://denny-hwang.github.io/git-science-park/>
- 저장소: <https://github.com/Denny-Hwang/git-science-park>

## 1. 마일스톤 (Milestones)

전체 개발은 6개 Phase, 총 10주로 구성한다. 각 Phase는 카테고리 단위로 묶어 디자인 일관성과 리뷰 효율을 확보한다.

| Phase | 기간 | 목표 |
|-------|------|------|
| 1 | Week 1 | 문서화, 허브 페이지, 파일럿 3개 실험 |
| 2 | Week 2-3 | 01-ancient (4), 02-scientific-revolution (7) |
| 3 | Week 4-5 | 03-precision-era (5), 04-energy-field (7) |
| 4 | Week 6-7 | 05-atomic (9), 06-quantum (7) |
| 5 | Week 8-9 | 07-relativity (7), 08-modern (7) |
| 6 | Week 10 | QA, 문서 최종화, v1.0 릴리스 |

### Phase별 상세 산출물

- **Phase 1 — 기반 구축 (Week 1)**
  - `docs/` 기획·설계 문서 작성(PRD, ROADMAP, USER-STORIES, 아키텍처/디자인 가이드).
  - 허브 페이지 `src/index.html` 구현: 8개 카테고리 그리드, 53개 실험 카드, 검색/필터, 반응형 레이아웃.
  - 메타데이터 스키마 `src/data/experiments.json` 확정 및 53개 항목 등록(초기 `status: planned`).
  - 공통 자산(카테고리 색상 토큰, 3탭 템플릿, Canvas/SVG 유틸 패턴) 정립.
  - 파일럿 실험 3개 구현으로 템플릿 검증: **#1 에라토스테네스의 지구 둘레 측정**, **#2 아르키메데스의 부력 원리**, **#6 갈릴레오의 자유낙하**.
  - GitHub Actions 자동 배포 파이프라인 점검.

- **Phase 2 — 고대·과학혁명 (Week 2-3)**
  - `01-ancient` 4개 완성(파일럿으로 만든 #1, #2 포함, #3 지레·#4 천동설 vs 지동설 추가).
  - `02-scientific-revolution` 7개 완성(#5~#11).
  - 누적 11개 실험 `status: ready` 전환.

- **Phase 3 — 18세기·19세기 (Week 4-5)**
  - `03-precision-era` 5개(#12~#16), `04-energy-field` 7개(#17~#23) 완성.
  - 누적 23개. 이 시점에 **v0.5** 목표(25개)에 근접.

- **Phase 4 — 원자·양자 (Week 6-7)**
  - `05-atomic` 9개(#24~#32), `06-quantum` 7개(#33~#39) 완성.
  - 누적 39개. 가장 무거운 Phase이므로 P3 난이도 실험 일정 버퍼를 확보한다.

- **Phase 5 — 상대성·현대 (Week 8-9)**
  - `07-relativity` 7개(#40~#46), `08-modern` 7개(#47~#53) 완성.
  - 누적 53개(전 실험 구현 완료).

- **Phase 6 — QA & 릴리스 (Week 10)**
  - 크로스 브라우저 점검(Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ / IE 미지원).
  - 반응형(모바일/태블릿/데스크톱) 및 오프라인 실행 검증, 접근성·성능 점검.
  - 전 실험 3탭(🎮 인터랙티브 / 📚 원리 학습 / 📜 역사적 맥락) 콘텐츠 최종 교정.
  - 문서 최종화 후 **v1.0** 태깅 및 릴리스.

## 2. 실험별 우선순위 기준

53개 실험을 모두 동시에 만들 수 없으므로 **시각적 임팩트 × 직관성 × 구현 난이도**를 기준으로 P1/P2/P3 세 등급으로 분류한다. 같은 카테고리 안에서도 우선순위가 다를 수 있으며, 각 Phase 내부에서 P1 → P2 → P3 순으로 착수한다.

| 우선순위 | 의미 | 선정 기준 | 착수 시점 |
|---------|------|----------|----------|
| **P1** | 먼저 | 시각적 임팩트가 크고 직관적. 슬라이더 하나만 움직여도 변화가 즉시 보이고, 사전 지식 없이도 "조작 → 관찰"이 성립. 초기 데모·홍보 효과가 큼. | Phase 내 가장 먼저 |
| **P2** | 중간 | 보통 난이도. 한두 단계의 배경 설명이 필요하지만 인터랙션 설계가 명확. 표준적인 Canvas/SVG 시뮬레이션으로 구현 가능. | P1 이후 |
| **P3** | 나중 | 추상적이거나 구현이 복잡. 수식 비중이 크거나(맥스웰 방정식, 슈뢰딩거 방정식 등) 정밀한 수치 시뮬레이션·다물체 계산이 필요. 시각화 난이도가 높아 설계 시간이 더 필요. | Phase 후반 / 버퍼 활용 |

판단에 사용하는 보조 지표:

- **시각적 임팩트**: 화면에서 변화가 얼마나 즉각적이고 크게 드러나는가.
- **직관성**: 배경 지식 없이도 조작-결과의 인과가 바로 이해되는가.
- **구현 복잡도**: 필요한 수학/물리 모델, Canvas 렌더링 부하, 상태 관리의 복잡성.
- **교육적 핵심도**: 과학사 서사에서 차지하는 비중(동률일 때 가산점).

## 3. 버전 계획 (Release Plan)

점진적 공개를 통해 초기부터 사용 가능한 결과물을 배포하고, 단계별로 실험 수를 늘려간다.

| 버전 | 범위 | 누적 실험 수 | 대응 Phase | 비고 |
|------|------|------------|-----------|------|
| **v0.1** | 허브 페이지 + 5개 실험 | 5 | Phase 1 ~ Phase 2 초반 | 템플릿·배포 파이프라인 검증, 첫 공개 데모 |
| **v0.5** | 절반 분량 실험 | 25 | Phase 3 완료 전후 | 전반부 카테고리 대부분 가용, 사용자 피드백 수집 |
| **v1.0** | 전체 실험 완성 | 53 | Phase 6 | 53개 실험 + 문서 최종화 + QA 완료, 정식 릴리스 |

- **v0.1**: 허브에서 53개 카드가 모두 보이되 미구현 항목은 `status: planned`로 "준비중" 비활성 표시. 구현 완료 5개만 `status: ready`.
- **v0.5**: 25개 `ready`. 나머지는 계속 "준비중". 이 시점에 사용자 피드백을 수집해 후반 Phase 설계에 반영.
- **v1.0**: 전 실험 `ready`. 모든 카테고리·전 난이도 커버, 오프라인 실행 및 크로스 브라우저 검증 완료.

## 5. 진행 추적 (Progress Tracking)

- 진행 상태는 `src/data/experiments.json`의 `status` 필드로 관리한다: `planned`(준비중) → `in-progress`(작업 중) → `ready`(완료).
- 허브 `src/index.html`는 `status` 값을 읽어 카드 표시를 제어한다(`planned`/`in-progress`는 "준비중" 비활성 표시, `ready`만 링크 활성화).
- 각 실험 페이지는 `src/experiments/{category}/{slug}.html` 경로 규칙을 따르며, 허브에서는 `./experiments/{category}/{slug}.html`로 링크한다.
- 실험 완료 정의(Definition of Done): 3탭(🎮 인터랙티브 / 📚 원리 학습 / 📜 역사적 맥락) 콘텐츠 완비, 단일 HTML 파일로 독립 실행 가능, Zero Dependencies 준수, 4개 지원 브라우저에서 동작 확인, 반응형 레이아웃 검증.
