# 사용자 스토리 (User Stories)

본 문서는 **Git Science Park**(과학사 인터랙티브 실험실)의 사용자 스토리를 정의한다. Git Science Park는 과학사의 핵심 실험 **53개**를 *"조작 → 관찰 → 해석"* 의 흐름으로 직접 다루며 배우는 인터랙티브 학습 플랫폼이다. 모든 실험은 **8개 카테고리**(고대 / 과학혁명 / 18세기 / 19세기 / 원자의 시대 / 양자역학 / 상대성·우주론 / 현대 물리학)로 분류되며, 각 실험 페이지는 **3개 탭**(🎮 인터랙티브 / 📚 원리 학습 / 📜 역사적 맥락)으로 구성된다.

이 문서의 사용자 스토리는 v1(초기 릴리스) 범위를 기준으로 작성되었으며, 각 스토리는 "As a ~, I want ~, so that ~" 형식과 수용 기준(Acceptance Criteria)을 포함한다.

---

## 1. 페르소나 (Personas)

Git Science Park의 핵심 사용자를 세 명의 페르소나로 정의한다. 인증·로그인 기능이 없는 v1 특성상 모든 사용자는 **익명(anonymous) 방문자**이며, 진입 장벽 없이 곧바로 학습을 시작한다.

| 페르소나 | 약칭 | 한 줄 정의 | 주요 동기 | 사용 맥락 |
|---|---|---|---|---|
| **고등학생** 지원 | P1 | 물리·과학 교과 개념을 시각적으로 이해하고 싶은 학습자 | 시험 대비, 추상 개념의 직관적 이해 | 주로 모바일, 짧은 세션, 자투리 시간 |
| **대학(원)생** 서연 | P2 | 전공 수업/과제와 연계해 원리를 깊이 파고드는 연구 지향 학습자 | 정량적 이해, 파라미터 탐구, 출처 확인 | 데스크톱 위주, 긴 세션, 집중 탐구 |
| **일반 과학 애호가** 민호 | P3 | 직업과 무관하게 과학사를 즐기는 호기심 많은 성인 | 지적 즐거움, 과학사 스토리텔링 | 모바일/태블릿 혼용, 오프라인·이동 중 |

### 1.1 페르소나 상세

#### P1 — 고등학생 지원 (만 17세)

- **배경**: 물리 I/II를 수강 중. 교과서의 정적인 그림과 수식만으로는 개념이 잘 와닿지 않는다.
- **목표**: "에라토스테네스의 지구 둘레 측정"이나 "이중 슬릿 실험" 같은 핵심 실험을 직접 조작하며 직관적으로 이해하고, 시험 직전 빠르게 복습한다.
- **기기/환경**: 스마트폰이 주력. 데이터 절약과 빠른 로딩을 중시한다.
- **좌절 지점**: 어려운 수식, 영어 위주 자료, 느린 페이지, 회원가입 강요.

#### P2 — 대학(원)생 서연 (만 23세)

- **배경**: 물리학·과학사 관련 전공. 강의 보조 자료나 발표 준비, 과제 해결을 위해 신뢰할 수 있는 시뮬레이션을 찾는다.
- **목표**: 파라미터를 바꿔가며 결과의 변화를 정량적으로 관찰하고, 원리(수식·유도)와 역사적 맥락(원전·연표)을 함께 확인한다.
- **기기/환경**: 데스크톱·노트북에서 큰 화면으로 집중. 여러 실험을 비교 탐색한다.
- **좌절 지점**: 출처 불명확한 정보, 얕은 설명, 모바일 전용으로 정보가 잘린 UI.

#### P3 — 일반 과학 애호가 민호 (만 35세)

- **배경**: IT 직군이지만 과학·역사 다큐와 책을 즐긴다. 전문 지식보다 "왜 중요한 실험인가"의 서사에 끌린다.
- **목표**: 출퇴근·여행 중 가볍게 둘러보며 과학사의 결정적 순간을 체험한다. 인터넷이 불안정한 환경에서도 보고 싶다.
- **기기/환경**: 지하철·기내 등 오프라인 상황. 태블릿과 스마트폰을 오간다.
- **좌절 지점**: 네트워크 끊김 시 동작 불가, 지나치게 학술적인 어조, 무거운 외부 의존성.

## 2. 사용자 스토리

> 형식: **As a ~, I want ~, so that ~** (한국어 풀이 + 형식 유지)
> 각 스토리에는 수용 기준(Acceptance Criteria, AC)을 2~3개씩 명시한다.

### 2.1 P1 — 고등학생 지원

#### US-P1-01 · 카테고리로 둘러보기

- **As a** 고등학생으로서, **I want** 8개 카테고리(고대~현대 물리학)로 정리된 실험 목록을 허브에서 한눈에 훑어보기를, **so that** 지금 배우는 단원과 관련된 실험을 빠르게 찾을 수 있다.
- **Acceptance Criteria**
  - [ ] 허브(`src/index.html`)에 8개 카테고리가 고유 색상·이모지·한글/영문 제목과 함께 표시된다.
  - [ ] 각 카테고리에 소속 실험 개수가 표기된다(고대 4 / 과학혁명 7 / 18세기 5 / 19세기 7 / 원자의 시대 9 / 양자역학 7 / 상대성·우주론 7 / 현대 물리학 7, 합계 53).
  - [ ] 카테고리 선택 시 해당 실험만 필터링되어 목록이 즉시 갱신된다.

#### US-P1-02 · 키워드로 실험 검색

- **As a** 고등학생으로서, **I want** 실험 이름·키워드로 검색하기를, **so that** "이중 슬릿"이나 "갈릴레이"처럼 떠오르는 단어로 곧장 원하는 실험에 닿을 수 있다.
- **Acceptance Criteria**
  - [ ] 허브 상단의 검색창에 입력하면 입력 중(실시간)에 결과가 좁혀진다.
  - [ ] 한글 제목, 영문 제목/슬러그, 핵심 키워드가 검색 대상에 포함된다.
  - [ ] 일치하는 실험이 없으면 "결과 없음" 안내가 표시된다.

#### US-P1-03 · 인터랙티브로 직접 조작

- **As a** 고등학생으로서, **I want** 실험의 🎮 인터랙티브 탭에서 슬라이더·버튼으로 변수를 직접 바꾸기를, **so that** 수식이 아니라 손으로 만지며 개념을 직관적으로 이해할 수 있다.
- **Acceptance Criteria**
  - [ ] 실험 페이지 진입 시 기본 탭이 🎮 인터랙티브이며 Canvas/SVG 시뮬레이션이 즉시 렌더링된다.
  - [ ] 입력값을 변경하면 시뮬레이션과 관찰값이 지연 없이(체감 60fps 수준) 반영된다.
  - [ ] "초기화" 동작으로 기본 상태로 되돌릴 수 있다.

#### US-P1-04 · 난이도 보고 고르기

- **As a** 고등학생으로서, **I want** 각 실험의 난이도(⭐1~⭐⭐⭐⭐⭐5)를 미리 확인하기를, **so that** 내 수준에 맞는 실험부터 부담 없이 시작할 수 있다.
- **Acceptance Criteria**
  - [ ] 허브의 실험 카드와 실험 페이지 헤더에 난이도가 별(⭐)로 표시된다.
  - [ ] 난이도 값은 `src/data/experiments.json`의 `difficulty`(1~5)와 일치한다.
  - [ ] 난이도 기준 정렬 또는 필터가 가능하다.

#### US-P1-05 · 모바일에서 쾌적하게

- **As a** 고등학생으로서, **I want** 스마트폰 화면에서도 허브와 실험이 깨지지 않고 편하게 동작하기를, **so that** 등하굣길 자투리 시간에 복습할 수 있다.
- **Acceptance Criteria**
  - [ ] 360px 폭 화면에서 가로 스크롤 없이 레이아웃이 정상 표시된다(반응형).
  - [ ] 슬라이더·버튼 등 조작 요소가 터치로 정확히 다뤄진다(터치 타깃 충분).
  - [ ] 시뮬레이션 Canvas가 화면 폭에 맞춰 비율을 유지하며 리사이즈된다.

### 2.2 P2 — 대학(원)생 서연

#### US-P2-01 · 파라미터 정량 탐구

- **As a** 대학(원)생으로서, **I want** 인터랙티브 탭에서 변수를 정밀하게 입력하고 결과 수치를 읽기를, **so that** 변수와 결과의 관계를 정량적으로 분석할 수 있다.
- **Acceptance Criteria**
  - [ ] 슬라이더와 함께 직접 수치 입력 또는 현재값 표시가 제공된다.
  - [ ] 관찰 결과(측정값/그래프)가 단위와 함께 명확히 표시된다.
  - [ ] 극단값 입력 시에도 시뮬레이션이 오류 없이 안정적으로 동작한다.

#### US-P2-02 · 원리(수식·유도) 학습

- **As a** 대학(원)생으로서, **I want** 📚 원리 학습 탭에서 핵심 수식과 유도 과정을 확인하기를, **so that** 인터랙티브로 본 현상을 이론적으로 뒷받침할 수 있다.
- **Acceptance Criteria**
  - [ ] 📚 원리 학습 탭에 핵심 개념·수식·결과 해석이 단계적으로 정리되어 있다.
  - [ ] 수식과 인터랙티브 탭의 변수 기호/의미가 일관된다.
  - [ ] 탭 전환 시 인터랙티브에서 조작한 상태가 불필요하게 초기화되지 않는다.

#### US-P2-03 · 역사적 맥락과 출처 확인

- **As a** 대학(원)생으로서, **I want** 📜 역사적 맥락 탭에서 실험의 배경·인물·연도·의의를 확인하기를, **so that** 발표·과제에서 신뢰할 수 있게 인용할 수 있다.
- **Acceptance Criteria**
  - [ ] 📜 역사적 맥락 탭에 실험자, 시기, 당대 맥락, 후대 영향이 서술된다.
  - [ ] 8개 카테고리의 시대 구분(고대~현대 물리학)과 내용이 모순되지 않는다.
  - [ ] 인물·연도 등 사실 정보가 본문 곳곳에서 일관되게 유지된다.

#### US-P2-04 · 실험 간 비교 탐색

- **As a** 대학(원)생으로서, **I want** 허브로 쉽게 돌아가 다른 실험으로 이동하기를, **so that** 관련 실험들을 비교하며 주제를 깊이 파고들 수 있다.
- **Acceptance Criteria**
  - [ ] 모든 실험 페이지에 허브로 돌아가는 명확한 내비게이션이 있다.
  - [ ] 같은 카테고리의 다른 실험으로 빠르게 이동할 수 있는 동선이 제공된다.
  - [ ] 링크 경로가 `./experiments/{category}/{slug}.html` 규칙을 따른다.

#### US-P2-05 · 데스크톱 대화면 활용

- **As a** 대학(원)생으로서, **I want** 큰 화면에서 시뮬레이션과 설명을 넓게 보기를, **so that** 디테일을 놓치지 않고 집중해 분석할 수 있다.
- **Acceptance Criteria**
  - [ ] 데스크톱 폭에서 Canvas/SVG가 충분히 크게 렌더링된다.
  - [ ] 본문·표·그래프가 잘리지 않고 가독성 있게 배치된다.
  - [ ] 창 크기 변경 시 레이아웃과 시뮬레이션이 자연스럽게 재배치된다.

#### US-P2-06 · 준비중 실험의 상태 인지

- **As a** 대학(원)생으로서, **I want** 아직 구현되지 않은 실험이 "준비중"으로 명확히 구분되기를, **so that** 어떤 실험을 지금 학습할 수 있는지 헷갈리지 않는다.
- **Acceptance Criteria**
  - [ ] `status=planned` 실험은 허브에서 "준비중"으로 비활성 표시된다.
  - [ ] 준비중 실험 카드는 클릭 시 빈 페이지로 이동하지 않는다(비활성/안내 처리).
  - [ ] 상태 표기는 `experiments.json`의 `status`(ready | planned | in-progress)와 일치한다.

### 2.3 P3 — 일반 과학 애호가 민호

#### US-P3-01 · 가입 없이 즉시 시작

- **As a** 과학 애호가로서, **I want** 로그인이나 가입 없이 곧바로 실험을 둘러보기를, **so that** 마찰 없이 호기심을 채울 수 있다.
- **Acceptance Criteria**
  - [ ] 허브 진입과 실험 사용에 어떠한 인증·가입 절차도 요구되지 않는다.
  - [ ] 첫 진입부터 실험을 보고 조작하기까지의 단계가 최소화되어 있다.
  - [ ] 개인정보 입력 없이 모든 v1 기능을 사용할 수 있다.

#### US-P3-02 · 오프라인에서 동작

- **As a** 과학 애호가로서, **I want** 인터넷이 불안정하거나 끊긴 상황에서도 실험이 동작하기를, **so that** 지하철·기내 같은 환경에서도 학습을 이어갈 수 있다.
- **Acceptance Criteria**
  - [ ] 각 실험은 독립 실행 가능한 단일 HTML 파일로, 외부 라이브러리/CDN 의존이 없다(Zero Dependencies).
  - [ ] 한 번 로드한 실험 페이지는 네트워크 없이도 시뮬레이션이 정상 동작한다.
  - [ ] 외부 폰트/스크립트 미로딩으로 인해 핵심 기능이 깨지지 않는다.

#### US-P3-03 · 스토리텔링으로 즐기기

- **As a** 과학 애호가로서, **I want** 어려운 수식보다 "왜 중요한 실험인가"의 이야기를 먼저 만나기를, **so that** 전문 지식 없이도 과학사의 결정적 순간을 즐길 수 있다.
- **Acceptance Criteria**
  - [ ] 📜 역사적 맥락 탭이 일반 독자도 이해할 서사 중심으로 서술된다.
  - [ ] 본문은 한국어로 쉽게 풀어 쓰되, 고유명사·식별자는 영문을 유지한다.
  - [ ] 3개 탭을 자유롭게 오가며 흥미에 따라 깊이를 조절할 수 있다.

#### US-P3-04 · 태블릿·모바일 혼용

- **As a** 과학 애호가로서, **I want** 태블릿과 스마트폰을 오가며 같은 경험을 누리기를, **so that** 기기를 바꿔도 끊김 없이 둘러볼 수 있다.
- **Acceptance Criteria**
  - [ ] 태블릿/모바일 양쪽에서 반응형 레이아웃이 일관되게 동작한다.
  - [ ] 세로/가로 회전 시 레이아웃과 시뮬레이션이 적절히 재배치된다.
  - [ ] 터치 제스처(탭·드래그)로 핵심 조작이 가능하다.

#### US-P3-05 · 가볍고 빠른 로딩

- **As a** 과학 애호가로서, **I want** 페이지가 빠르게 뜨고 데이터를 적게 쓰기를, **so that** 이동 중 모바일 데이터로도 부담 없이 즐길 수 있다.
- **Acceptance Criteria**
  - [ ] 외부 프레임워크/빌드 산출물 없이 순수 정적 자원만 로드한다.
  - [ ] 허브와 실험 페이지가 체감상 빠르게 초기 렌더링된다.
  - [ ] 불필요한 대용량 에셋 없이 핵심 콘텐츠가 우선 표시된다.

### 2.4 공통/접근성 스토리 (All Personas)

#### US-CMN-01 · 키보드만으로 탐색

- **As a** 사용자로서(특히 마우스 사용이 어려운 경우), **I want** 키보드만으로 허브 탐색과 탭 전환을 하기를, **so that** 입력 수단에 관계없이 학습할 수 있다.
- **Acceptance Criteria**
  - [ ] 검색·필터·실험 링크·탭이 Tab/Enter 등 키보드로 조작된다.
  - [ ] 포커스 표시(focus indicator)가 시각적으로 드러난다.
  - [ ] 탭 컨트롤이 의미 있는 순서로 포커스를 받는다.

#### US-CMN-02 · 스크린리더 친화

- **As a** 시각 보조 기술 사용자로서, **I want** 화면 요소가 스크린리더로 의미 있게 읽히기를, **so that** 콘텐츠 구조를 이해할 수 있다.
- **Acceptance Criteria**
  - [ ] 제목 계층(h1~), 랜드마크, 버튼/탭에 적절한 레이블/ARIA 속성이 부여된다.
  - [ ] 이모지·아이콘만으로 의미를 전달하지 않고 텍스트 대안을 함께 제공한다.
  - [ ] 탭 전환 시 활성 탭 상태가 보조 기술에 전달된다.

#### US-CMN-03 · 색상에 의존하지 않는 구분

- **As a** 색각 이상 사용자로서, **I want** 카테고리/상태가 색 외의 단서로도 구분되기를, **so that** 색상만으로 판별하지 않아도 된다.
- **Acceptance Criteria**
  - [ ] 카테고리는 색상과 함께 이모지·텍스트 라벨로 구분된다.
  - [ ] "준비중" 등 상태가 색 외에 텍스트/아이콘으로도 표시된다.
  - [ ] 주요 텍스트/배경 대비가 충분히 확보된다.

### 3.1 v1 비범위 (Won't / this release)

다음 항목은 v1에서 **의도적으로 제외**한다. 이는 향후 릴리스에서 재검토될 수 있으나, v1의 단순함·정적 실행·무마찰 진입 원칙을 지키기 위해 포함하지 않는다.

- **인증/로그인**: 모든 사용자는 익명으로 접근하며 개인 계정·세션이 없다.
- **진행률 저장 / 즐겨찾기**: 학습 이력, 완료 표시, 북마크 등 상태 영속화를 제공하지 않는다(서버·DB 없음).
- **다국어(i18n)**: 본문은 한국어를 기준으로 하며 다국어 전환 UI를 제공하지 않는다(코드/식별자/고유명사는 영문 유지).
